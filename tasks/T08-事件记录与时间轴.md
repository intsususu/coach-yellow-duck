# T08 · 事件记录与时间轴（E1 + E2）

> 依赖：T07 ｜ PRD 引用：§5.7 E1、§5.8 E2、§4.2 事件色板、§6.1 模型、§9.3

## 目标
实现事件模块：记录/编辑弹窗（E2）+ 事件时间轴（E1），并**打通全局＋记事件入口**——把 T02 的占位 sheet 替换为真实表单。保存后写入 `appState.events`，各页图表叠加随之更新，触发 Toast。

## 范围
**做：** E1 时间轴、E2 记录弹窗、全局入口接线、本机持久化（SwiftData/Core Data 或先内存+UserDefaults）。
**不做：** HealthKit（T09）。

## 详细要求

### E2 · 记录 / 编辑弹窗（全局 sheet）
- 顶部：取消 / 标题「新建事件」(编辑时「编辑事件」) / 保存。
- **类型选择**：生病 / 损伤 / 饮酒 / 旅行（/ 其他），选中用类型色（§4.2）。
- **持续切换**：单日 / 时间段（选时间段时显示开始 + 结束两个日期）。
- **日期**：默认 `2026-06-18`，日历选择。
- **标题**（可选，空则取类型 label）。
- **备注**：占位「添加描述，例如"感冒发烧，停训"…」。
- 提示文案「记录后会在体重/睡眠/运动图表上标注，帮你解释数据波动。」
- **保存逻辑**（PRD 原型）：构造 `HealthEvent`（`isPeriod = 选了时间段`）→ 插入 `appState.events` 顶部 → 关闭 → `showToast("已记录：{标题}")`（2.2s）→ 持久化。

### E1 · 事件时间轴
- 标题「事件」+ 右上`＋`（→ E2 新建）。
- **类型筛选 chips**：全部 / 生病 / 损伤 / 饮酒 / 旅行。
- 按月份分组（如「2026 年 6 月」）。
- **时间轴**：每条 = 类型色点 + 标题 + 单日/时间段标签 + 日期(段含起止与天数) + 备注。点击可进入 E2 编辑。
- 初始数据 = `appState.events`（T01 的 4 条 mock）。

### 接线
- 把 T02 `presentEventEditor()` 的占位 sheet 换成真实 E2。
- T07「＋记录特殊事件」入口 → E1 时间轴（或直接 E2，按 PRD）。
- 各页（总览近期事件、体重/睡眠/运动事件叠加）自动反映新增事件（因共享 `appState.events`）。

### 持久化
- 用户新建的事件需重启后仍在；mock 初始 4 条可作为种子。

## 交付文件
- `HealthApp/Features/Events/EventTimelineView.swift`（E1）
- `HealthApp/Features/Events/EventEditorView.swift`（E2）
- `HealthApp/Repository/EventStore.swift`（持久化）
- 修改 T02 的 `presentEventEditor` 接线。

## 验收标准
- [ ] 任意 Tab ＋ → 弹真实记录表单；类型/单日·时间段/日期/标题/备注 可填。
- [ ] 保存后：列表顶部出现新事件，Toast「已记录：…」显示并消失。
- [ ] 新事件即时出现在总览近期事件、体重/睡眠/运动图表叠加中。
- [ ] E1 时间轴：筛选 chips、按月分组、单日/时间段显示正确，可点进编辑。
- [ ] 新建事件重启后仍存在。
