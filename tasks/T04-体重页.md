# T04 · 体重页（A2 + E3 事件叠加）

> 依赖：T03 ｜ PRD 引用：§5.2、§8 图表与分析、§6 数据

## 目标
实现体重 Tab：长期折线趋势 + 目标线 + 时间范围切换 + 统计 + 最近记录 + **事件叠加（E3 展示）**。这是核心页。

## 范围
**做：** A2 全部 + E3 的图表叠加与图例、事件影响条、关联洞察。建立可复用的 `WeightChart` 组件（含事件叠加），供后续页面参考。
**不做：** 事件的新增/编辑（T08）；本页只是**展示** `appState.events`。

## 详细要求
1. **顶部**：标题「体重」+ 右上`＋`。
2. **时间范围切换**：周/月/年/全部（分段控件），切换重绘，数据取仓库 W/M/Y（"全部"用 Y 或合并）。
3. **主折线图**（Swift Charts，建在 `Charts/WeightChart.swift`）：
   - `LineMark` + `AreaMark` 渐变填充（`brandBlue`）。
   - **目标线**：`RuleMark` 虚线 = `goalWeight`，标注「目标 73」。
   - **事件叠加**（§8.3）：开关「在图上显示事件」；单日事件 = 点标记（`◆`/`PointMark`，类型色），时间段事件 = 色带（`RectangleMark`，类型 bg 半透明）。
   - 图例：生病 / 损伤(段) / 饮酒 / 旅行(段)。
4. **事件影响条**：如「5月31日 · 感冒发烧 — 病后一周运动暂停，体重回升 0.6kg。点查看 ›」。
5. **统计三元组**：当前 `77.1` / 累计 `-13.9` / 历史最低 `71.9`（另可显示历史最高 `91.2`）。
6. **最近记录列表**：6月15日 77.1kg / 6月08日 78.0kg / 6月01日 78.9kg …（取 W 序列倒序）。
7. **关联洞察**（E3 底部）：「拉伤那周运动暂停，体重回升 0.6kg；出差期间作息乱、下降也停滞。」

## 关键实现点
- 目标线与统计用 `goalWeight`（来自 `AppState`，T07 可改），不写死。
- 事件叠加读 `appState.events`，按 `isPeriod` 分流为点/带。

## 交付文件
- `HealthApp/Features/Weight/WeightView.swift`（替换占位）
- `HealthApp/Charts/WeightChart.swift`
- 如需：`Weight/WeightViewModel.swift`
