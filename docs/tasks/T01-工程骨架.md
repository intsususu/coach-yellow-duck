# T01 · 工程骨架

> 依赖：无 ｜ PRD 引用：§1.4 技术栈、§4 设计规范、§6 数据模型、§9 架构

## 目标
搭好可编译运行的空壳 iOS 工程：目录结构、设计系统、数据模型、Mock 仓库。完成后 App 能启动（即使只是一个空白页），且后续页面所需的数据、颜色、类型都已就绪。

## 范围
**做：**
- 新建 SwiftUI iOS 工程（iOS 16.0+，名称 `HealthApp`，Bundle Id 占位）。
- 建立 §9.2 的目录结构。
- 设计系统：颜色 token、卡片/圆环/Pill 等基础复用组件（可先空实现/简化）。
- 全部数据模型（§6.1）。
- `HealthDataRepository` 协议 + `MockHealthRepository`（内置 §6.2 全部 mock 数据）。

**不做：**
- 任何具体页面 UI（占位页留给 T02）。
- HealthKit、持久化（留给 T08/T09）。

## 详细要求

### 1. 目录结构（§9.2）
```
HealthApp/
├── App/                // @main App 入口
├── Models/             // 数据模型
├── Repository/         // 协议 + Mock
├── Features/           // 各页面（本任务先建空目录）
├── Charts/             // 图表组件（本任务先建空目录）
├── DesignSystem/       // 颜色/组件
└── Resources/          // mock json（可选，或直接代码内置）
```

### 2. 设计系统（DesignSystem/）
- `Color+Tokens.swift`：实现 PRD §4.1 与 §4.2 全部颜色（`brandBlue #2563eb`、`sleepIndigo #4f46e5`、`exerciseOrange #ea580c`、`successGreen #16a34a`、文字色、hairline，以及事件 5 类的 color/bg）。用 `Color(hex:)` 扩展。
- 基础组件（可先最简实现，后续页面复用）：`CardView`（白底圆角 16、轻阴影/描边）、`RingMetric`（环形进度 + 数值 + 标签）、`PillButton`、`SectionTitle`。

### 3. 数据模型（Models/）
按 PRD §6.1 实现：`WeightSample`、`SleepSample`、`ExerciseSample`、`EventType`（含 label + color/bg 映射）、`HealthEvent`（单日/时间段，`isPeriod`）。另加：
- `enum TimeRange { case week, month, year, all }`（睡眠/运动按需子集）。

### 4. 仓库（Repository/）
- `protocol HealthDataRepository`（§9.1）：`weightSeries / sleepSeries / exerciseSeries / events / saveEvent`，async。
- `MockHealthRepository`：内置 PRD §6.2 的**全部真实数组**（体重 W/M/Y、睡眠 14 晚、运动 6 月、初始 4 条事件）。日期字符串解析为 `Date`。`saveEvent` 先存内存数组。

### 5. App 入口（App/）
- `@main` App，`WindowGroup` 暂时显示一个写着 "HealthApp" 的占位 `Text`（T02 替换为 TabView）。
- 通过环境注入 `MockHealthRepository`（`@StateObject` 或 `environmentObject`），方便 T09 切换。

## 交付文件（约定，后续任务沿用）

## 验收标准
- [ ] 工程可编译、可在模拟器启动并显示占位文字。
- [ ] 颜色 token、事件色板与 PRD §4 完全一致。
- [ ] 4 个数据模型 + 仓库协议齐全。
- [ ] `MockHealthRepository` 返回 PRD §6.2 原值（抽查体重 W 首尾、事件 4 条）。
- [ ] 仓库以协议形式注入环境，便于替换。
